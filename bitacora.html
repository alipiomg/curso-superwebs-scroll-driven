⚡ Decisiones clave

HTML autocontenido sin build tools
Todo en un solo HTML con CSS y JS inline. Deploy directo en GitHub Pages con npx serve.
Porque: minimiza friccion para el alumno. Cero dependencias, cero configuracion. Copiar, abrir y funciona.
localStorage para gamificacion zero-backend
XP, badges, progreso y retos se persisten en localStorage del navegador.
Porque: no hay backend ni base de datos. El curso es 100% estatico. El alumno no necesita cuenta.
course.json como unica fuente de verdad
Metadatos, rutas, prerequisitos y gamificacion definidos en un solo JSON. Los HTML lo consumen.
Porque: facilita validacion cruzada y evita inconsistencias entre landing, curso y conceptos.
2 rutas pedagogicas: lineal + exploratoria
Ruta lineal para principiantes (prerequisitos estrictos). Ruta exploratoria para avanzados (solo requiere leccion 1).
Porque: diferentes perfiles de alumno necesitan diferentes niveles de estructura.

⚠️ Gotchas detectados

HTML tags dentro de inline code se renderizan como elementos reales
Fix: La funcion mdToHtml() debe escapar HTML (escapeHtml()) dentro de backticks inline y bloques de codigo ANTES de envolverlos en <code>. Sin esto, `<canvas>` se renderiza como un canvas HTML real.
</script> dentro de JSON embebido rompe el parser HTML
Fix: Escapar como <\\/script> en cualquier string JSON que contenga tags de cierre de script.
Badges no se recalculan al recargar la pagina
Fix: Llamar checkBadges(true) (modo silencioso) en init() antes de renderBadges(). Sin esto, si el estado pierde badges, no se recuperan hasta completar otra accion.
IDs de prerequisitos deben validarse contra course.json
Fix: Antes de generar artefactos, verificar que cada ID de prerequisito en los concept files exista en course.json.
